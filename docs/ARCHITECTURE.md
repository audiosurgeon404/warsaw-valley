# ARCHITECTURE — как устроен код

Ваниль + ES-модули. Три слоя из `CLAUDE.md` отражены в коде: **данные → мир → рендер**,
плюс тонкая **ретро-обёртка** вокруг.

---

## 1. Поток данных (сверху вниз)

```
AirMQ GraphQL  ──fetch──►  airmq.js  ──normalize──►  world.js  ──state──►  render (map.js)
 (docs/API.md)             (сеть,        (маппинг       (чистая        (canvas + DOM HUD)
                            кеш,          данные→мир)    логика)
                            таймаут)
        ▲                                                    │
        └──────── localStorage cache (офлайн, D-31) ─────────┘
```

Принцип: **строго однонаправленно**. Сеть не рисует. Рендер не ходит в сеть. Между ними —
`world.js`, который превращает сырые сенсорные значения в «состояние мира» (погода, смог,
присутствие персонажей).

## 2. Модули `src/js/`

| Файл            | Ответственность | Экспорт (черновой контракт) |
|-----------------|-----------------|------------------------------|
| `main.js`       | Точка входа. Грузит конфиг, стартует опрос, крутит цикл рендера. | — |
| `airmq.js`      | Весь сетевой слой. GraphQL-запросы, таймаут, retry, кеш. Никто больше не знает про GraphQL. | `getCharacterState(id)`, `getManyStates(ids)` |
| `world.js`      | Чистые функции: данные → мир. Пороги смога, палитра дня, присутствие. | `deriveWeather(state)`, `smogLevel(pm25)`, `dayPalette(temp, online)` |
| `characters.js` | Загрузка `data/characters.json`, склейка «персонаж + его состояние». | `loadCharacters()`, `withState(chars, states)` |
| `map.js`        | Рендер сцены на canvas: тайлы, спрайты, погодные оверлеи, клики. | `initScene(canvas)`, `drawFrame(worldState)` |
| `ui.js`         | Ретро-обёртка: титул-бар, marquee, счётчик, карточка персонажа, гостевая. | `mountChrome()`, `showCharacterCard(char)` |

> Контракты чернового вида — уточняются на Этапе 1–3. Главное правило (D-23): сеть только в `airmq.js`.

## 3. Форма «состояния персонажа» (нормализованная)

`airmq.js` отдаёт наверх такой объект (никаких сырых GraphQL-полей дальше не течёт):

```js
{
  id: "PL010010258",
  name: "Fiodar H",
  online: true,
  time: "2026-07-21T10:11:37.985Z", // UTC, из сенсора
  metrics: {
    temp: 16.92,   // °C   (null если нет)
    hum: 68.27,    // %
    press: null,   // hPa
    pm1: 0, pm25: 1, pm10: 2, // µg/m³
    count: null,   // µSv/h
    nox: null, voc: null,
    aqi: null
  },
  stale: false     // true, если это данные из кеша, а сеть не ответила
}
```

`world.js` дальше превращает это в `worldState` для рендера:

```js
{
  daylight: "warm-day",   // из temp
  fog: 0.1,               // из hum
  smog: "clean",          // из pm25
  characters: [{ id, name, x, y, sprite, mood, present }]
}
```

## 4. Циклы

- **Таймер данных** (`setInterval`, дефолт 60 c, D-30): дёргает `airmq.getManyStates`,
  кладёт в кеш, обновляет `worldState`.
- **Кадр рендера** (`requestAnimationFrame`, D-33): рисует текущий `worldState` и анимации.
  Кадр НЕ ходит в сеть и НЕ пересчитывает пороги — только рисует.

Разделение таймера и кадра — чтобы анимация была плавной (60fps), а данные — редкими.

## 5. Конфигурация и секреты

- `config/env.example.js` — шаблон (в git). `config/env.local.js` — реальные значения (gitignored).
- Импорт токена и настроек — только в `airmq.js`. Остальной код настроек не касается.

```js
// config/env.local.js (пример, НЕ коммитить)
export const AIRMQ_TOKEN = "eyJ...";      // секрет
export const AIRMQ_ENDPOINT = "https://api-app.airmq.cc/graphql";
export const POLL_INTERVAL_MS = 60000;
```

## 6. Сборка и запуск

- Dev: **Vite** (`npm run dev`) — быстрый сервер + ESM из коробки.
- Prod: `npm run build` → статические файлы в `dist/` → любой статик-хостинг.
- Прокси токена для прода — отдельная мини-функция (см. `docs/API.md`), не часть фронта.

## 7. Тестируемость

Логика мира (`world.js`) — чистые функции без DOM и сети → тривиально покрывается юнит-тестами.
Сеть (`airmq.js`) — за фасадом, легко мокается. Рендер тестируем глазами (это игрушка).

# CHARACTERS — как заводить персонажей

Персонаж = датчик AirMQ + личность + место на карте + спрайт. Единственный источник
правды — `src/data/characters.json` (директива D-2). Код имена не хардкодит.

---

## 1. Схема одного персонажа

```jsonc
{
  "id": "PL010010258",        // AirMQ _id датчика (обязателен, связь с данными)
  "name": "Fiodar H",          // имя как в AirMQ (важно для совпадения)
  "displayName": "Фёдар",      // как показываем в мире (можно кириллицей)
  "role": "farmer",            // роль в мире: farmer | blacksmith | fisher | ...
  "bio": "Хозяин долины у центра. Любит ясное небо.", // короткое описание
  "home": { "x": 12, "y": 8 }, // клетка домика на карте (в тайлах)
  "sprite": "char_fiodar",     // базовое имя спрайт-листа в assets/sprites
  "palette": "warm",           // визуальный акцент персонажа
  "geo": { "lat": 52.2136, "lon": 21.0217 } // реальные координаты (для карты Варшавы)
}
```

Поля `id`, `name`, `home`, `sprite` — обязательны. Остальное — вкус и атмосфера.

## 2. Как добавить нового персонажа

1. Найди датчик в AirMQ:
   ```graphql
   query { locations(filter: { city: "Warsaw" }) { _id name isOnline metricList } }
   ```
2. Возьми `_id` и точное `name`.
3. Добавь объект в массив `src/data/characters.json`.
4. Положи спрайт-лист `assets/sprites/<sprite>_idle.png` (и `_walk/_sleep/_wave`).
5. Готово — код подхватит его сам (D-2). Ничего в JS менять не нужно.

## 3. Стартовый каст

| id             | name         | Роль        | Заметка                                   |
|----------------|--------------|-------------|-------------------------------------------|
| `PL010010258`  | Fiodar H     | farmer      | Онлайн, полный набор PM+Temp+Hum.         |
| `PL010010395`  | Dwoch Mieczy | blacksmith  | Недавно перепрошит — данные бывают пустые.|

> Правильное написание — **«Dwoch Mieczy»** (в AirMQ именно так, не "meczy"/"meczów").

## 4. Настроение и присутствие (как оживает персонаж)

Логика — в `world.js`, значения — из живых метрик:

- **present**: `online === true` и есть свежий `currentValue` → персонаж на карте.
  Иначе — «спит дома» (`Zzz`), карточка «отдыхает».
- **mood** (сердечко/выражение): считаем по воздуху вокруг него —
  чисто (PM2.5 низкий, комфортная температура) → счастлив ☺;
  смог/жара/холод → хмурится ☹; нет данных → нейтрально/спит.
- **action-анимация**: `idle` по умолчанию, `walk` при лёгком блуждании, `wave` при клике,
  `sleep` при offline.

## 5. Канон-арт

Эталонные картинки живут в `src/assets/reference/`. По ним рисуем спрайты и тайлы
(палитра, детали, композиция). Референс не обязан быть пиксель-перфект.

### Dwoch Mieczy — `assets/reference/dwoch_mieczy.png`

Двор «Двух Мечей» = **усадьба-пара с тремя питомцами** под закатом.

**Сцена / тайлы:**
- Двухэтажный кремово-белый дом, бурая двускатная крыша, тёплый свет фонаря у крыльца.
- Небо — закат: сиренево-розовый верх → оранжевое зарево у горизонта, тонкий **полумесяц**.
- Зелёный **кованый забор** с калиткой, каменная (брусчатка) дорожка к дому.
- Деревья: **апельсиновое** (оранжевые плоды) слева, **вишнёвое/с красными ягодами** справа, тёмная хвоя на фоне.

**Персонажи (спрайты `char_dwoch_her` / `char_dwoch_him`):**
- Она: светлые длинные волосы, **бирюзовая куртка**, розовая лямка сумки, тёмные шорты, красные кроссовки.
- Он: серая **шапка-бини**, борода, чёрная футболка с принтом, тёмные шорты, часы.

**Питомцы:**
- `pet_bernese` — бернский зенненхунд (чёрно-бело-рыжий), рядом с хозяевами.
- `pet_cat_tux` — чёрно-белый кот (tuxedo), стоит у дорожки.
- `pet_cat_ginger` — рыжий табби, дремлет у забора.

**Палитра-акцент персонажа:** `sunset` (тёплое золото заката + бирюзовый акцент куртки).
При отрисовке держим целочисленный масштаб и палитру из `docs/DESIGN.md` (D-10/D-11).

> Как это связано с данными: дом «Dwoch Mieczy» на карте — этот двор. `online` →
> хозяева и питомцы во дворе (idle); `offline`/нет данных → закрытая калитка, «Zzz».
> Высокий PM2.5 → лёгкий смог поверх тёплого заката.

### Fiodar H — `assets/reference/fiodar_h.png`

Дом «Fiodar H» = **окно под крышей варшавской каменицы**, ночь. Городской, «ночной» контраст к закатной усадьбе Dwoch Mieczy.

**Сцена / тайлы:**
- Классическая **каменица** (доходный дом): серо-бежевая штукатурка, лепнина, карнизы, пилястры/полуколонны, ряды окон; внизу намёк на витрины первого этажа.
- Небо — глубокая синь ночи, **полная луна**.
- Крыша жестяная, с **антеннами и спутниковыми тарелками**; на коньке — голуби и чайка.
- Верхнее **распахнутое окно** светится тёплым оранжевым.

**Интерьер окна (тёплый свет):** настольная лампа, книжные полки, картины на стене — уют.

**Персонаж (спрайт `char_fiodar`):**
- Кудрявые тёмные волосы, **оливковый свитер**, облокотился на подоконник.

**Питомец:**
- `pet_cat_ginger` — рыжий кот, свернулся на подоконнике.

**Палитра-акцент персонажа:** `night-window` (тёмно-синяя ночь + тёплое оранжевое окно).

> Как это связано с данными: дом «Fiodar H» — эта каменица. `online` → в окне горит
> тёплый свет, Fiodar у подоконника; `offline`/нет данных → окно тёмное, «Zzz».
> Высокий PM2.5 → дымка/гало вокруг луны и фонарей.

## 6. Приватность

Координаты датчиков — публичные в AirMQ, но всё равно: не добавляй персонажей на основе
чужих датчиков без желания их владельцев (D-53). Стартовый каст — «свои».

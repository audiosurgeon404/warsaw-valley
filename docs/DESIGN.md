# DESIGN — визуальный язык Warsaw Valley

Как этот мир выглядит и звучит. Два эстетических слоя, наложенных друг на друга:
**Stardew Valley** (игровая сцена) и **веб нулевых** (обёртка сайта).

---

## 1. Настроение

- **Cozy, тёплый, ламповый.** Осеннее утро в польской долине, но это Варшава.
- **Ностальгия нулевых**: сайт будто собран в 2004-м на коленке, с любовью и marquee.
- **Живой**: мир дышит вместе с данными — теплеет, туманится, засыпает.

## 3. Пиксель-арт

- **Базовый тайл**: 16×16 px. Спрайт персонажа: 16×24 px (голова+тело).
- **Масштаб только целочисленный**: рендерим ×3 (итог тайл 48px). Меняем множитель — не дробим.
- CSS обязателен:
  ```css
  .pixel { image-rendering: pixelated; image-rendering: crisp-edges; }
  ```
- Спрайт-листы: горизонтальные полосы кадров, кадр = 16×24, imгена `char_<name>_<action>.png`
  (`idle`, `walk`, `sleep`, `wave`).
- Анимация: 2–4 кадра, ~6–8 fps. Idle — лёгкое «дыхание».

## 4. Шрифты

- **Игровая зона / HUD / имена персонажей** — пиксельный шрифт (например «Press Start 2P»
  или локальный bitmap-шрифт). Кладём локально в `src/assets`, не тянем из CDN в проде.
- **Длинные тексты** (доки в UI, гостевая) — обычный читаемый шрифт (Verdana/Tahoma —
  как раз дух нулевых и хорошо читается).
- Размеры пиксельного шрифта — кратные пикселю (8/16/24), иначе «замыливается».

## 5. Ретро-приёмы нулевых (дозированно, со вкусом)

Разрешено и приветствуется:
- **Бевелы** (`border` в стиле outset/inset) вокруг панелей и кнопок.
- **Титул-бары** синие с белым текстом, как окна Windows.
- **Счётчик посещений** (визуальный, локальный) — «You are visitor #00042».
- **`<marquee>`-подобная строка** новостей мира (реализуем CSS/JS, уважая reduced-motion).
- **Гостевая книга** (локальная, в localStorage) — «оставь привет персонажам».
- **Under construction** гифка-пасхалка в подвале.
- **Табличная эстетика** как приём (но реальная раскладка — Grid/Flex).

Запрещено (перебор): автоплей музыки, мигание с эпилептической частотой, всплывающие окна,
курсор-хвосты поверх игровой зоны (можно как отключаемая пасхалка).

## 6. Как данные превращаются в картинку (маппинги мира)

Чистые функции (см. `world.js`, директива D-25):

- **Температура → палитра дня**
  - `< 0°C` → морозное небо + снег; `0–12` → прохладное утро; `12–22` → тёплый день;
    `> 22` → закатное золото; ночь/`offline` → ночная палитра.
- **Влажность → дымка**
  - `> 80%` → лёгкий туман по низу; `> 90%` → капли на «стекле» окна сайта.
- **PM2.5 → смог** (порог по шкале воздуха)
  - `0–5` чисто; `5–15` лёгкая дымка; `15–35` заметный смог; `> 35` плотный смог + кашляющий персонаж.
- **online → присутствие**
  - `online` — персонаж на карте, анимация idle/walk;
  - `offline`/нет данных — домик закрыт, «Zzz», карточка «персонаж отдыхает».

Точные пороги — константы в `world.js`, чтобы легко крутить.

## 8. Ассеты (что нарисовать)

- `tiles/`: трава, тень травы, дорожка, вода, мост через Вислу, дерево, куст.
- `sprites/`: `char_fiodar_*`, `char_dwoch_*` (idle/walk/sleep/wave), домики.
- `audio/`: спокойный луп (ферма-эмбиент), клик-звук, «пробуждение» персонажа.
- `ui/`: бевел-рамки, иконки погоды (☀🌧🌫❄), сердечко-настроение, курсор.

Плейсхолдеры на старте — простые цветные прямоугольники из палитры, потом заменяем артом.
